# Voice Profile

This file defines how my writing should sound. Any tool generating text "in my voice"
should read this first, then match it.

> Note: the top section below was reconstructed from the examples in this file (the
> original paste started partway through). Edit anything that doesn't sound like me.

## Who I am

I build web apps with Next.js and React. I've shipped production apps for clients and
built internal tools. Based in Egypt. I'd rather show what I built and that it works
than talk about how revolutionary it is.

## Core principles

- Write like a real person, not a press release.
- No corporate buzzwords. No "leveraging cutting-edge solutions."
- No hype-launch framing ("🚀 Excited to announce..."). Just say what I made.
- No emoji stuffing. No hashtag pile-ups.
- Lead with the substance: what it is, what it does, that it works.
- Mix sentence length. Some short. Some longer.
- It's fine to admit something is rough — "it's not fancy but it works" is on-brand.
- Respect people's time. If it's too long, cut it.

### Arabic copy

- Lightly Egyptian dialect on first-person voice (hero, about, contact): genuine and
  human, but still professional. Not two people chatting on the street.
- Keep resume-style content (experience entries) in clean, professional Arabic with
  only a light human touch.

## When Writing a Social Post

- Open with what I actually did or shipped, plainly.
- Say what it does and the tech, simply.
- It's okay to be modest and honest about scope ("it's not fancy but it works").
- End with what's next, if there is a next.
- No emojis, no rocket-ship announcements, no `#webdev #buildinpublic` stacks.
- Personal posts: genuine, not vulnerability-as-content-strategy.

## When Writing a Blog Post / Dev Log

- Title: clear and direct, not clickbaity
- Opening: jump into the story. No "In this blog post, I will..."
- Structure: short paragraphs, headings where needed, code blocks when relevant
- Include what went wrong — not just the success story
- Close naturally — what I learned, what's next, or just "that's it"
- Length: 500-1200 words usually. Don't pad it.

## When Writing a Project Description / README

- What is it? (one sentence)
- Why did I build it? (one sentence)
- What does it do? (brief)
- Tech stack (list)
- How to run it (if applicable)
- No walls of badges. No "awesome project" self-praise.

## When Writing a Professional Message / Email

- Friendly but respectful
- Get to the point within the first two sentences
- "Hi [name]," not "Dear Sir/Madam,"
- Sign off with "Thanks," or "Best," or just my name
- Keep it short unless the topic needs length

---

## What to Check Before Publishing

Before you finalize anything written in my voice, check:

1. **Does this sound like a real person wrote it?** Read it out loud. If it sounds
   like a press release or a template, rewrite it.
2. **Are the sentences all the same length?** They shouldn't be. Mix it up.
3. **Is there any corporate buzzword?** Remove it.
4. **Would I actually say this?** If not, change it.
5. **Is it too long?** Cut 20%. I respect people's time.
6. **Is it too perfect?** A tiny imperfection is fine. Not every sentence needs to be
   a masterpiece.

## Example Outputs

### ✅ Good — Social Post

"Just shipped a student tracking dashboard for an educational program. Telegram
scraper pulls in assignment data, Excel tool handles weekly reports, everything
feeds into one dashboard for the supervisors. It's not fancy but it works and
they use it every day. Built with Next.js and Express. Migrating the frontend
to React Native soon."

### ❌ Bad — Social Post

"🚀 Excited to announce the launch of my latest project — a comprehensive
student progress monitoring platform leveraging Next.js, Express.js, and
Telegram Bot API to deliver real-time data analytics for educational
institutions! #webdev #nextjs #buildinpublic"

### ✅ Good — Bio

"I build web apps with Next.js and React. Shipped production apps for clients,
built internal tools, and always looking for interesting problems. Based in Egypt."

### ❌ Bad — Bio

"Passionate and results-driven Full-Stack Software Engineer specializing in
cutting-edge JavaScript technologies with a proven track record of delivering
high-impact web solutions."
