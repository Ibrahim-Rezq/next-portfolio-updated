بناء موقع شخصي يبدو بسيطاً على الورق، لكنه يحمل دقائق كثيرة في التنفيذ. إليك ما تعلّمته من بناء هذا الموقع.

## لماذا Next.js 16

الموقع كان يحتاج أن يكون سريعاً، وصديقاً لمحركات البحث، وثنائي اللغة (عربي وإنجليزي). Next.js 16 مع App Router أعطاني التصيير من جهة الخادم تلقائياً، وبنية المسار `[locale]` مع **next-intl** جعلت معمارية ثنائي اللغة واضحة ومنطقية.

```tsx
// كل صفحة تحصل على اللغة من الرابط: /en/ أو /ar/
export default async function LocaleLayout({ params }) {
  const { locale } = await params;
  return (
    <html lang={locale} dir={locale === "ar" ? "rtl" : "ltr"}>
      ...
    </html>
  );
}
```

هذا النمط الواحد — `<html lang dir>` يُعيَّن من جهة الخادم — هو الأساس الكامل لقصة SEO ثنائي اللغة. غوغل تفهرس `/en/` و `/ar/` كصفحتين مستقلتين.

## نظام التصميم

اشتغلت من وثيقة تسليم تصميم حدّدت نسبة الألوان 60/30/10:

- **60% كريمي** (`#FEF8EC`) — قاعدة الخلفية
- **30% تيل غامق** (`#134E52`) — البنية والعناوين
- **10% ذهبي دافئ** (`#C4943F`) — اللمسات والأزرار

كل هذا يعيش في CSS custom properties في `globals.css`، ومكشوف لـ Tailwind v4 عبر `@theme inline { ... }`. النتيجة: كلاسات مثل `text-text-heading` و `bg-gold-500` تعني شيئاً حقيقياً بدل قيم hex خام.

## الزخارف الهندسية الإسلامية

الأنماط الهندسية — النجمة الثمانية، الشبكة الماسية، التكرار الهندسي — مرسومة كـ SVG و PNG في `public/patterns/`. تظهر بشفافية 5–20% لإضافة الدفء دون ضوضاء بصرية.

نمط الـ hero عبارة عن SVG متحرك فعلي يرسم نفسه عند التحميل ويستجيب لحركة الماوس. لا canvas، فقط SVG paths و `requestAnimationFrame`.

## ما كنت سأفعله بشكل مختلف

**أبدأ بـ design tokens أولاً.** أعدت بناء طبقة CSS custom properties مرتين لأنني لم ألتزم بالتسمية الدلالية مبكراً. الذهاب مباشرة إلى `--text-heading` بدل `--teal-700` من اليوم الأول كان سيوفّر عليّ إعادة البناء.

**أستخدم next-intl من البداية.** بدأ الموقع بـ locale provider من جهة العميل، مما يعني أن الخادم كان يُصيّر HTML الإنجليزي دائماً بغض النظر عن تفضيل المستخدم — سيء لـ SEO. الترحيل إلى توجيه قائم على URL كان القرار الصحيح، لكنه كان أثقل وسط البناء مما كان سيكون عليه في البداية.
